Dev Handover Checklist

Client: _______________ Date: _______________

How to use this checklist

Walk through each section with the designer to confirm what's ready, what's missing, and what's out of scope. Click checkboxes to mark items complete.

  • Required — must have before build can start
  • Optional — nice to have, can add if time permits
  • Out of scope — not included in this quote
Budget: $750 — This is a template-based build (Framer/Squarespace/similar). Keep scope tight: 1–5 pages, minimal custom code, no complex interactions. Anything beyond basic template customisation will need a scope change.
1

Project Info

Basic project details and timeline expectations.

Project name / brand confirmed
What's the business/brand name exactly as it should appear?
Website URL / domain
Does the client already own a domain? What is it?
Launch deadline
Is there a hard deadline? Event, campaign, etc?
Platform confirmed Required
Where will this be built? Framer, Squarespace, Webflow, etc.
2

Pages & Structure

What pages are being built? Keep it minimal for this budget.

Page list confirmed Required
List every page. For $750, recommend max 5 pages.
Navigation structure
What goes in the main nav? Any dropdowns? (Dropdowns add complexity)
Footer content
What's in the footer? Links, social icons, newsletter signup?
3

Design Files

What design assets are being handed over?

Design file access Required
Figma link, or exported files. Dev needs edit/view access.
Desktop design complete Required
All pages designed at desktop width (1440px or 1280px typical).
Mobile design complete Required
All pages designed at mobile width (375px typical). Not optional!
If no mobile designs, dev will interpret — may not match expectations.
Tablet design Optional
768px breakpoint. Often skipped — dev interpolates between mobile/desktop.
Hover states defined Optional
What happens when you hover on buttons, links, cards?
Component states
If there are forms: empty, filled, error, success states?
4

Assets

Logo, images, fonts — everything needed to build.

Logo files Required
SVG preferred. PNG with transparent background as backup. Need both light and dark versions?
Logo — SVG format
Logo — PNG fallback (@2x)
Favicon (square version of logo or mark)
Fonts Required
Font names + where to get them. Google Fonts? Adobe Fonts? Custom files?
Images — final, print-ready Required
All images used in design, exported at 2x, optimised for web (under 500kb each ideally).
Stock images: confirm licensing. Client's own photos: confirm they have rights.
Icons
SVG format. Or specify icon library (e.g., Feather, Phosphor, Heroicons).
5

Content

Is the copy final? This is the #1 cause of delays.

All copy is FINAL Required
Headlines, body copy, button labels — all approved by client. No "we'll update this later".
Placeholder copy = guaranteed revision rounds = scope creep.
Copy is in the design file
Or provided in a separate doc with clear page/section labels.
Legal pages Optional
Privacy Policy, Terms & Conditions. Client responsibility to provide, not designer/dev.
SEO basics
Page titles, meta descriptions. Can be added later but good to have upfront.
6

Functionality

What needs to actually DO something? Each item here adds complexity.

Contact form
Where do submissions go? Email address? What fields are needed?
External links
Social media links, booking system, etc. Need actual URLs.
Newsletter signup Optional
Which provider? Mailchimp, Klaviyo, etc. Need account access or embed code.
E-commerce Out of Scope
Selling products? Not included at this budget. Would need separate quote.
Blog / CMS Out of Scope
If client needs to update content themselves, that's CMS setup — adds complexity.
Custom animations Out of Scope
Scroll animations, complex transitions, parallax. Template defaults only at this budget.
7

Launch Requirements

What's needed to actually go live?

Domain ownership confirmed
Who owns the domain? Client needs to own it or purchase it.
Domain access / DNS
Login details for domain registrar (GoDaddy, Namecheap, etc) — needed to connect domain.
Hosting plan
Most template platforms include hosting (Framer, Squarespace). Confirm who pays ongoing.
Analytics Optional
Google Analytics? Need client's GA4 property ID, or set up new one.
SSL certificate
Usually included with modern platforms. Just confirm HTTPS will work.
8

Handover Notes

Anything else the dev needs to know?

Design decisions / rationale
Any specific spacing, interaction, or layout notes for dev?
Things that are "close enough" vs "pixel perfect"
Where can dev use template defaults vs. where does it need to match exactly?
Revision expectations
How many revision rounds included? What counts as a revision vs. new feature?
Client contact
Does dev communicate with client directly, or through designer?
