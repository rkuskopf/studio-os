# Web Project — Dev Handover Checklist

A reusable checklist for handing over a completed website to a client. Use at end of project, or adapt sections for kick-off discovery.

---

## 1. Pre-Launch: Content & Copy

- [ ] All pages have final approved copy (no placeholder text)
- [ ] All images are high-res, optimised for web (< 200kb where possible)
- [ ] Favicon set
- [ ] Browser tab title / page titles set correctly
- [ ] 404 page exists and is styled
- [ ] All internal links work
- [ ] All external links open in new tab
- [ ] Contact form tested — submissions land in correct inbox
- [ ] Any embedded maps, videos, or third-party widgets tested

---

## 2. Pre-Launch: SEO & Meta

- [ ] Page titles and meta descriptions set for all key pages
- [ ] OG image set (for social sharing previews)
- [ ] robots.txt in place
- [ ] sitemap.xml generated and submitted to Google Search Console
- [ ] Google Analytics / GA4 installed and firing
- [ ] Google Search Console verified

---

## 3. Pre-Launch: Performance

- [ ] Images compressed (use Squoosh or similar)
- [ ] Google PageSpeed score > 80 on mobile
- [ ] No broken images or missing assets
- [ ] Fonts loading correctly (no FOUT on slow connections)
- [ ] Tested on Chrome, Safari, Firefox
- [ ] Tested on iPhone (Safari) and Android (Chrome)
- [ ] Tested at 320px, 768px, 1280px, 1440px breakpoints

---

## 4. Pre-Launch: Accessibility

- [ ] Colour contrast passes WCAG AA (use [Colour Contrast Checker](https://webaim.org/resources/contrastchecker/))
- [ ] All images have alt text
- [ ] All form inputs have labels
- [ ] Site is keyboard-navigable
- [ ] No auto-playing video/audio

---

## 5. Domain & Hosting

- [ ] Domain purchased and confirmed (registrar: ______)
- [ ] DNS pointed to hosting/platform
- [ ] SSL certificate active (https://)
- [ ] www → non-www redirect (or vice versa) confirmed
- [ ] Old/staging URL redirects to live URL

---

## 6. Access Credentials — Handover to Client

Hand over in a secure password manager share (1Password, Bitwarden) or encrypted doc. Never send passwords in plain text.

---

## 7. Client Training

- [ ] Walkthrough recorded (Loom or similar) — how to edit content
- [ ] How to add/edit pages
- [ ] How to update images
- [ ] How to publish changes
- [ ] Where to find form submissions
- [ ] What NOT to touch (if anything)
- [ ] Support contact: how to reach Rowan after handover

---

## 8. Backup & Maintenance

- [ ] Backup taken before go-live
- [ ] Client knows how to take a backup (or auto-backup is set up)
- [ ] Hosting renewal date noted (put in client file)
- [ ] Domain renewal date noted
- [ ] Any recurring costs documented (hosting, plugins, fonts, etc.)
- [ ] Post-launch support period agreed (e.g. 2 weeks of bug fixes included)

---

## 9. Final Sign-Off

- [ ] Client has reviewed and approved the live site
- [ ] Invoice issued / payment received
- [ ] Project file archived in `clients/[name]/`
- [ ] Case study notes captured (for portfolio — see `clients/ux-portfolio.md`)
